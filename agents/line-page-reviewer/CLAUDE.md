# 🔍 LINEページ 反映漏れチェックエージェント

新しいお寺のLINEページが正しく実装されているかを自動検査するエージェント。
静蓮寺（joren-ji）のベースから複製したページを対象とし、
旧寺院情報の残存・未更新箇所・リンク切れを検出して報告する。

---

## 起動方法

以下のように呼びかける：

```
{寺院スラッグ}のページをレビューしてください。
寺院情報：
  - 正式名称：○○寺
  - 住所：〒xxx-xxxx ○○県...
  - 電話：xx-xxxx-xxxx
  - LINE URL：https://...
  - LIFF URL：https://...
```

---

## チェック手順

### Step 1: 対象ファイルをすべて読み込む

以下のファイルを順にReadする：
1. `{slug}/houji/houji.html`
2. `{slug}/houji/list.html`
3. `{slug}/houji/flow.html`
4. `{slug}/event.html`
5. `{slug}/chat-introduction.html`

### Step 2: 残存文字列スキャン

以下の文字列がいずれかのファイルに残っていないか全文検索する：

```
静蓮寺
jorenji
103pffyg
5XPp6PB
lin.ee/5XPp6PB
RGS1oQ
03-3843-4034
03-3843-3442
zen0801key
下谷1-12-21
110-0004
正定寺
03-3841-1853
```

残存が見つかった場合：
```
❌ 残存あり："{文字列}" が {ファイル名} に残っています
```

見つからない場合：
```
✅ 残存なし："{文字列}"
```

### Step 3: 必須項目チェック

提供された寺院情報をもとに、以下が正しく反映されているか確認する：

#### houji/houji.html
- [ ] `<title>` に正しい寺院名が入っているか
- [ ] ヒーロー画像 `src` が `../images/{slug}_hondo.jpg` になっているか
- [ ] LIFF URLが新しいものになっているか
- [ ] フッターの正式名称・住所・電話番号が正しいか

#### houji/list.html
- [ ] 塔婆申込みの FAX番号・メールアドレスが正しいか
- [ ] 連絡先セクションの寺院名・電話番号が正しいか
- [ ] `静蓮寺` `正定寺` の文字列が残っていないか

#### houji/flow.html
- [ ] Googleマップ iframe の `src` に正しい住所が入っているか
- [ ] 「Googleマップで開く」の `href` が正しいか
- [ ] 住所・電話番号が正しいか

#### event.html
- [ ] カルーセル画像3枚のパスが `./images/{slug}_carousel_{n}.jpg` になっているか
- [ ] `data-date` の年が当年（または翌年）になっているか
- [ ] 「静蓮寺」のテキストが残っていないか

#### chat-introduction.html
- [ ] LINEメッセージURLが3箇所すべて新しいものになっているか
- [ ] 住職画像 `src` が `./images/{slug}_priest.jpg` になっているか
- [ ] `静蓮寺へLINEで相談する` のテキストが更新されているか

### Step 4: `line://` スキーム検出

`line://` を含む href がないか確認する。見つかった場合はエラーとして報告。

### Step 5: 画像ファイル存在確認

以下のファイルが `{slug}/images/` ディレクトリに存在するか `ls` で確認する：
- `{slug}_hondo.jpg`（またはヒーロー画像）
- `{slug}_carousel_1.jpg`
- `{slug}_carousel_2.jpg`
- `{slug}_carousel_3.jpg`
- `{slug}_priest.jpg`

存在しないファイルは ❌ で報告する。

### Step 6: DESIGN_SPEC.md の存在確認

`{slug}/DESIGN_SPEC.md` が存在するか確認する。なければ作成を促す。

## 出力フォーマット

```
# 🔍 {寺院スラッグ} LINEページ レビュー結果
実施日：{今日の日付}

## ✅ 合格項目
- ...

## ❌ 要修正項目
- {ファイル名}：{問題の説明}
  → 修正方法：{具体的な対処}

## ⚠️ 確認が必要な項目
- ...

## 総合判定
🟢 リリース可 / 🔴 要修正あり（{件数}件）
```

---

## 自動修正モード

ユーザーが「修正も行ってください」と指定した場合、
検出した問題を自動的にEditツールで修正し、
修正後に再度チェックを走らせて 0件 になることを確認する。

---

## 注意事項

- チェックはあくまで文字列レベルの検査。目視確認（実機でのLINE内表示）は必ず行うこと。
- `data-date` が過去日になっている場合は警告するが、行事の有無はお寺に確認が必要。
- お布施・費用情報の妥当性はチェックしない（お寺の判断に委ねる）。
